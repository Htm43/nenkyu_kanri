
# 年休管理ノート

ブラウザで使うシンプルな年休管理アプリです。JSON ファイルを開いて、年休の取得履歴や設定をそのまま編集できます。

## このアプリの取得方法

### 方法1: zipをダウンロードして使う（Git不要）

1. 配布された zip ファイルをダウンロード
2. 任意のフォルダに解凍
3. 解凍されたフォルダ内の `Nenkyu.html` を開く

### 方法2: Gitでクローン（Git使用者向け）

```bash
git clone <リポジトリURL>
cd <フォルダ名>
```

その後 `Nenkyu.html` を開く

### 方法3: ファイルを直接コピー

以下のファイルを同一フォルダに配置

* `Nenkyu.html`
* `Nenkyu.css`
* `Nenkyu.js`

## できること

* JSON ファイルを開いて年休データを編集
* 前回開いていたファイルを復元
* 全休 / 時間休の記録追加・編集・削除
* 各記録にメモを残して、あとから理由を見返す
* 残り時間、残り日数、期限までの日数の確認
* 月別の取得時間の可視化
* 昼休憩控除を含む時間休計算

## ファイル構成

* `Nenkyu.html`: 画面のマークアップ
* `Nenkyu.css`: レイアウトとスタイル
* `Nenkyu.js`: 状態管理、描画、入力処理、ファイル操作
* `nenkyuu.json`: サンプルデータ

## 使い方

1. `Nenkyu.html` を Edge / Chrome などで開く
2. 初期画面で `ファイルを開く`、`新規作成`、または `前回のデータを復元` を選ぶ
3. 記録や設定を編集する
4. 保存は JSON ファイルへ自動反映される

## ローカルサーバーで開く方法

`file://` で不安定な場合は以下の方法を使用

### 方法A: Python

```bash
cd フォルダパス
python -m http.server 8000
```

ブラウザで `http://localhost:8000` を開く

### 方法B: Node.js

```bash
npx serve
```

表示されたURLを開く

### 方法C: VSCode

* 拡張機能「Live Server」をインストール
* `Nenkyu.html` を右クリック → 「Open with Live Server」

## データ形式

現在の保存形式は次のような構造です。

```json
{
  "version": 2,
  "settings": {
    "totalDays": 11,
    "hoursPerDay": 7,
    "deadline": "2027-03-31",
    "deductLunchBreak": true,
    "lunchStart": "12:00",
    "lunchEnd": "12:45"
  },
  "entries": [
    {
      "id": "uuid",
      "date": "2026-04-22",
      "type": "partial",
      "startTime": "15:30",
      "endTime": "16:15",
      "note": "病院の予約"
    }
  ]
}
```

* `version`: データ形式のバージョン
* `settings`: 年休計算に使う設定
* `entries`: 取得履歴
* `note`: 各記録の補足メモ

補足:

* 旧形式の `records` 配列も読み込み可能
* `type` は `full` または `partial`
* `full` の場合は `startTime` / `endTime` は `null` 可

## 保存と復元

* File System Access API を使って JSON を直接読み書き
* 前回開いたファイルのハンドルは IndexedDB に保存
* 復元時はブラウザの権限状態によって再許可が必要

## 注意

* `file://` での動作が不安定な場合あり
* Chromium系ブラウザ推奨

## 技術選定理由

### Web（HTML/CSS/JS）でUIを構築

* ブラウザのみで動作（インストール不要）
* OS非依存
* 配布がファイル単位で完結
* UI修正が即時反映

### ローカルJSONを直接操作

* サーバー不要
* データがローカルに閉じる
* ファイル単位でバックアップ・共有可能
* 構造が単純で可読性が高い

## この構成の利点

* 静的ファイルのみで動作
* 外部サービス・認証不要
* オフライン利用可能
* JSON単体でデータ管理可能
* 配布・更新が容易

## 制約

* 同時編集不可
* デバイス間同期なし
* 自動バックアップなし
* 大規模データ非対応

## 他技術を選定した場合

### クラウド + DB

* サーバー運用・認証が必要
* ネットワーク依存
* コスト発生
* 複数端末同期・共有が可能

### RDB + APIサーバー

* バックエンド実装が必要
* スキーマ管理・運用負荷
* 大規模データ・複雑集計に対応

### デスクトップアプリ

* ビルド・配布が必要
* アプリサイズ増加
* ネイティブ機能利用可

### IndexedDBのみ

* ブラウザごとにデータ分離
* ファイルとして外部管理しづらい
* 操作は簡略化可能

## 要約

個人の年休管理用途に適する構成。

* 単一ユーザー前提
* 小規模データ
* 外部共有不要
* プライバシー重視

結果:

* セットアップ不要
* 運用コストなし
* データ管理が単純

結論:

ローカル完結型の個人データ管理として適合する。
