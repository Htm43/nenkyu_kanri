# 年休管理ノート

ブラウザで使うシンプルな年休管理アプリです。JSON ファイルを開いて、年休の取得履歴や設定をそのまま編集できます。

## できること

- JSON ファイルを開いて年休データを編集
- 前回開いていたファイルを復元
- 全休 / 時間休の記録追加・編集・削除
- 各記録にメモを残して、あとから理由を見返す
- 残り時間、残り日数、期限までの日数の確認
- 月別の取得時間の可視化
- 昼休憩控除を含む時間休計算

## 構成

### ブラウザ版（GitHub Pages）

PCブラウザから使うメインのアプリです。

- `index.html`: 画面のマークアップ
- `Nenkyu.css`: レイアウトとスタイル
- `Nenkyu.js`: 状態管理、描画、入力処理、ファイル操作

### モバイルアプリ版（Flutter / 別リポジトリで開発中）

スマホブラウザでは File System Access API が使えないため、Flutter による別アプリとして開発しています。
同じ JSON ファイルを読み書きします。

- Android 版：[Htm43/nenkyu-flutter](https://github.com/Htm43/nenkyu-flutter)（非公開・開発中）
- iOS 版：開発予定

## 使い方

### ブラウザ版

1. [nenkyu-web](https://htm43.github.io/nenkyu-web/) をブラウザで開く
2. 初期画面で `ファイルを開く`、`新規作成`、または `前回のデータを復元` を選ぶ
3. 記録や設定を編集する
4. 保存は JSON ファイルへ自動反映される

**データの置き場所について**

JSON ファイルをクラウドストレージの同期フォルダに置くことで、複数の PC 間でデータを共有できます。
- OneDrive、Google Drive、Dropbox などのデスクトップアプリを入れてローカルに同期しているフォルダであれば使えます
- ブラウザからクラウド上のファイルを直接読み込むことはできません

## データ形式

ブラウザ版・モバイルアプリ版で共通して使う形式です。

```json
{
  "version": 2,
  "settings": {
    "totalDays": 11,
    "hoursPerDay": 7,
    "deadline": "2027-03-31",
    "deductLunchBreak": true,
    "lunchStart": "12:00",
    "lunchEnd": "12:45"
  },
  "entries": [
    {
      "id": "uuid",
      "date": "2026-04-22",
      "type": "partial",
      "startTime": "15:30",
      "endTime": "16:15",
      "note": "病院の予約"
    }
  ]
}
```

- `version`: データ形式のバージョン
- `settings`: 年休計算に使う設定
- `entries`: 取得履歴
- `note`: 各記録の補足メモ

補足:

- 旧形式の `records` 配列も読み込み可能です
- `type` は `full` または `partial`
- `full` の場合は `startTime` / `endTime` は `null` でも構いません

## 保存と復元

- File System Access API を使って JSON を直接読み書きします
- 前回開いたファイルのハンドルは IndexedDB に保存されます
- 復元時はブラウザの権限状態によって再許可が必要です

## 注意

- File System Access API を使うため、対応ブラウザは主に Chromium 系（Chrome / Edge）です
- Safari・スマホブラウザには対応していません
- スマホからの利用は Flutter アプリ版をご利用ください（[Htm43/nenkyu-flutter](https://github.com/Htm43/nenkyu-flutter)）

## 技術選定理由

### Web + ローカルJSON

- ブラウザのみで動作（インストール不要）
- OS非依存
- サーバー不要
- データがローカルに閉じる
- JSONで明示的なデータ構造

## この構成の利点

- 静的ファイルのみで動作
- 外部サービス・認証不要
- オフライン利用可能
- ファイル単位でバックアップ・移動可能
- JSONをDropbox / OneDrive / Google Driveの同期フォルダに置くことで複数PC間での共有も可能

## 制約

- 同時編集不可
- スマホブラウザからの編集不可（File System Access API非対応のため）→ Flutter アプリ版で対応中
- 自動バックアップなし
- 大規模データ非対応

## 他技術との比較

### クラウド + DB

- サーバー・認証・運用が必要
- ネットワーク依存
- 同期・共有は可能

### RDB + API

- バックエンド実装が必要
- スキーマ管理が必要
- 大規模データ対応

### デスクトップアプリ

- ビルド・配布が必要
- 環境依存増加
- ネイティブ機能利用可

### Excel / VBA

- Excel環境が必須
- マクロ制限・警告あり
- UIがシート構造に制約される
- データ構造が暗黙的

## 要約

個人の年休管理用途に適する。

- 単一ユーザー
- 小規模データ
- 外部共有不要

結果:

- セットアップ不要
- 運用コストなし
- データ管理が単純

結論:

ローカル完結の個人用途として適合する。
クラウドストレージの同期フォルダを活用することで、デバイス間同期も無理なく実現できる。
スマホからの編集は別途 Flutter アプリ（[Htm43/nenkyu-flutter](https://github.com/Htm43/nenkyu-flutter)）で対応中。
