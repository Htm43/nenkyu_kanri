
# 年休管理ノート

ブラウザで使うシンプルな年休管理アプリです。JSON ファイルを開いて、年休の取得履歴や設定をそのまま編集できます。

## このアプリの取得方法

### 方法1: zipをダウンロードして使う（Git不要）

1. 配布された zip ファイルをダウンロード
2. 任意のフォルダに解凍
3. 解凍されたフォルダ内の `index.html` を開く

### 方法2: Gitでクローン（Git使用者向け）

```bash
git clone <リポジトリURL>
cd <フォルダ名>
```

その後 `index.html` を開く

### 方法3: ファイルを直接コピー

以下のファイルを同一フォルダに配置

* `index.html`
* `Nenkyu.css`
* `Nenkyu.js`

## できること

* JSON ファイルを開いて年休データを編集
* 前回開いていたファイルを復元
* 全休 / 時間休の記録追加・編集・削除
* 各記録にメモを残して、あとから理由を見返す
* 残り時間、残り日数、期限までの日数の確認
* 月別の取得時間の可視化
* 昼休憩控除を含む時間休計算

## ファイル構成

* `index.html`: 画面のマークアップ
* `Nenkyu.css`: レイアウトとスタイル
* `Nenkyu.js`: 状態管理、描画、入力処理、ファイル操作
* `nenkyuu.json`: サンプルデータ

## 使い方

1. `index.html` を Edge / Chrome などで開く
2. 初期画面で `ファイルを開く`、`新規作成`、または `前回のデータを復元` を選ぶ
3. 記録や設定を編集する
4. 保存は JSON ファイルへ自動反映される

## ローカルサーバーで開く方法

`file://` で不安定な場合は以下の方法を使用

### 方法A: Python

```bash
cd フォルダパス
python -m http.server 8000
```

ブラウザで `http://localhost:8000` を開く

### 方法B: Node.js

```bash
npx serve
```

表示されたURLを開く

### 方法C: VSCode

* 拡張機能「Live Server」をインストール
* `index.html` を右クリック → 「Open with Live Server」

## データ形式

```json
{
  "version": 2,
  "settings": {
    "totalDays": 11,
    "hoursPerDay": 7,
    "deadline": "2027-03-31",
    "deductLunchBreak": true,
    "lunchStart": "12:00",
    "lunchEnd": "12:45"
  },
  "entries": [
    {
      "id": "uuid",
      "date": "2026-04-22",
      "type": "partial",
      "startTime": "15:30",
      "endTime": "16:15",
      "note": "病院の予約"
    }
  ]
}
```

* `version`: データ形式のバージョン
* `settings`: 年休計算に使う設定
* `entries`: 取得履歴
* `note`: 補足メモ

補足:

* 旧形式 `records` も読み込み可能
* `type`: `full` / `partial`
* `full` の場合 `startTime` / `endTime` は `null` 可

## 保存と復元

* File System Access API で JSON を直接読み書き
* ファイルハンドルを IndexedDB に保存
* 復元時は再許可が必要な場合あり

## 注意

* `file://` は不安定な場合あり
* Chromium系ブラウザ推奨

## 技術選定理由

### Web + ローカルJSON

* ブラウザのみで動作（インストール不要）
* OS非依存
* サーバー不要
* データがローカルに閉じる
* JSONで明示的なデータ構造

## この構成の利点

* 静的ファイルのみで動作
* 外部サービス・認証不要
* オフライン利用可能
* ファイル単位でバックアップ・移動可能

## 制約

* 同時編集不可
* デバイス間同期なし
* 自動バックアップなし
* 大規模データ非対応

## 他技術との比較

### クラウド + DB

* サーバー・認証・運用が必要
* ネットワーク依存
* 同期・共有は可能

### RDB + API

* バックエンド実装が必要
* スキーマ管理が必要
* 大規模データ対応

### デスクトップアプリ

* ビルド・配布が必要
* 環境依存増加
* ネイティブ機能利用可

### Excel / VBA

* Excel環境が必須
* マクロ制限・警告あり
* UIがシート構造に制約される
* データ構造が暗黙的

## 要約

個人の年休管理用途に適する。

* 単一ユーザー
* 小規模データ
* 外部共有不要

結果:

* セットアップ不要
* 運用コストなし
* データ管理が単純

結論:

ローカル完結の個人用途として適合する。
