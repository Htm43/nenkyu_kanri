
# 年休管理アプリ　設計書

## 全体構成

```
アプリ本体     → GitHub Pages（静的ホスティング）
データ(JSON)  → 個人のOneDriveフォルダ
               形式は今のJSONのまま
```

---

## PCブラウザ版

**現状のまま、変更なし**

```
GitHub PagesのURLをブラウザで開く
↓
「ファイルを開く」でOneDrive内のJSONを選ぶ
↓
読み書き・自動保存（今と同じ）
↓
OneDriveが勝手にクラウド同期
```

---

## モバイルアプリ版（Android / iOS）

**新規開発**

```
GitHubのリリースページにAPKを置く（Android）
↓
直インストール
↓
OneDrive内の同じJSONを読み書き
↓
PCで編集したデータをスマホで確認・編集
```

### 技術構成

```
Flutterで書く（共通）
├── アプリ本体のUI
├── JSON読み書き
├── OneDrive連携
└── ビジネスロジック全部

ネイティブで書く（追加するだけ）
├── Androidウィジェット（Kotlin）
└── iOSウィジェット（Swift）
```

### 実装する機能

| 機能 | 優先度 |
|------|--------|
| OneDriveのJSONを開く | 必須 |
| サマリー表示（残り時間・日数・期限） | 必須 |
| 記録の追加・編集・削除 | 必須 |
| 前回ファイルの自動復元 | 必須 |
| ホーム画面ウィジェット | 後回し |

# 年休管理アプリ　設計書

## 全体構成

```
アプリ本体     → GitHub Pages（静的ホスティング）
データ(JSON)  → 個人のOneDrive/Dropboxフォルダ
               形式は今のJSONのまま
```

---

## PCブラウザ版

**現状のまま、変更なし**

```
GitHub PagesのURLをブラウザで開く
↓
「ファイルを開く」でOneDrive内のJSONを選ぶ
↓
読み書き・自動保存（今と同じ）
↓
OneDriveが勝手にクラウド同期
```

---

## モバイルアプリ版（Android / iOS）

**新規開発**

```
GitHubのリリースページにAPKを置く（Android）
↓
直インストール
↓
OneDrive/Dropbox内の同じJSONを読み書き
↓
PCで編集したデータをスマホで確認・編集
```

### 技術構成

```
Flutterで書く（共通）
├── アプリ本体のUI
├── ビジネスロジック全部
└── MethodChannel経由でファイル操作を呼ぶ

Kotlinで書く（Android）
├── SAF（Storage Access Framework）でファイル選択
├── URIを使ってJSONを読み込み
├── URIを使ってJSONを書き戻し
├── URIをSharedPreferencesに保存（前回ファイルの自動復元）
└── Androidウィジェット（後回し）

Swiftで書く（iOS・後回し）
└── iOSウィジェット
```

### MethodChannelの設計

```
Flutter → Kotlin
├── openFile()        → SAFでファイル選択ダイアログ → URIとJSON文字列を返す
├── saveFile(json)    → 保存済みURIにJSON文字列を書き戻す
└── restoreFile()     → 前回のURIからJSON文字列を読み込む
```

### 実装する機能

| 機能 | 優先度 | 実装場所 |
|------|--------|---------|
| OneDrive/DropboxのJSONを開く | 必須 | Kotlin |
| JSONを書き戻す | 必須 | Kotlin |
| 前回ファイルの自動復元 | 必須 | Kotlin |
| サマリー表示 | 必須 | Flutter |
| 記録の追加・編集・削除 | 必須 | Flutter |
| 設定画面 | 必須 | Flutter |
| ホーム画面ウィジェット | 後回し | Kotlin |
